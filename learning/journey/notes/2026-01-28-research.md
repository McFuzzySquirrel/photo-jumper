## What we discussed
Based on collected feedback and initial testing, there were definately quite a lot of gameplay issues and we came to the conlcusion that we needed to research the basic "rules" that need to be in place for a platform jumper game which need to be followed to make a platformer feel and behave as someone would expect.

So we discussed this at length, and I asked him to go do some research on what should be taken into consideration when designing and building a platform jumper, which he did and which we worked on this evening.

Since we are using AI to help with this, I explained the importance of context for an agent and that giving it rules to follow would help in ensuring that that it always keeps these rules in mind when we are looking to change or add anything. Better yet, it would be cool if we could also create an "enforcer" of these rules. Now there are many ways to go, but I (we) decided on the following plan:

- Take the research and create a copilot.instructions.md
- Create a custom agent system prompt to make sure rules have been followed.

## What we decided
- Create a copilot.instructions file that acts as the "law". See [here](.github/instructions/copilot.instructions.md)
